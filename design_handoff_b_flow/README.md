# Handoff: "오늘숙제" — 부모용 숙제 관리 플랫폼 (B 방향)

부모가 자녀의 숙제를 등록·검사하고 리워드를 지급하는 모바일 앱의 핵심 플로우 5화면 디자인입니다.
학습앱(Duolingo·Khan)의 밝고 가벼운 게이미피케이션 무드 + 학부모용의 차분한 신뢰감을 결합한 방향(B안)입니다.

## 📌 이 파일들에 대하여 (먼저 읽어주세요)

`code/` 안의 파일은 **HTML/React(JSX) 프로토타입 = 디자인 레퍼런스**입니다. 그대로 복붙해 출시하는 코드가 아니라, **여러분 코드베이스의 기존 환경(프레임워크·디자인시스템·컴포넌트)으로 이 디자인을 재현**하기 위한 정밀 참조물입니다.

- 이미 React/Vue/RN/Flutter 등 환경이 있다면 → 그 환경의 기존 컴포넌트·패턴으로 아래 스펙대로 재현하세요.
- 환경이 없다면 → 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.
- 본 프로토타입은 인라인 스타일 + 전역 CSS 토큰을 씁니다. 실제 구현에서는 토큰을 디자인시스템 변수로, 인라인 스타일을 컴포넌트/유틸리티 클래스로 옮기는 것을 권장합니다.

**Fidelity: High-fidelity.** 색·타이포·간격·radius·그림자까지 최종값입니다. 픽셀 단위로 재현하세요. (일러스트 자리표시자, 목록 항목 등 더미 데이터는 실제 데이터로 교체)

## 🔄 Interactions & Behavior
- **CTA/버튼:** 탭 시 약 0.96 scale + 그림자 약화 정도의 가벼운 press 피드백 권장. 트랜지션 120–180ms.
- **칩/세그먼트(과목·포인트):** 단일 선택 토글. 선택 시 채움색 전환.
- **검사 결과 "수정":** 해당 문항의 정답 처리(O/X)와 점수를 부모가 수동 보정 → 점수 히어로(숫자·링 %)·"오답 N·정답 N" 즉시 갱신.
- **리워드 "교환":** 잔액 ≥ 비용일 때만 활성. 교환 시 확인 다이얼로그 → 잔액 차감, 진행바 갱신. 부족분은 비활성+"부족" 표기.
- **네비게이션 플로우:** 로그인 → (자녀 연결) → 홈. 홈 "지금 검사하기"/완료 숙제 → 검사 결과. 홈 퀵액션 "숙제 입력" → 입력. 탭바 "리워드" → 리워드.
- **빈 상태:** 검사 대기 숙제 0개일 때 "검사할 숙제가 없어요" 류 빈 상태 필요(미디자인 — 동일 톤으로).
- **접근성:** 터치 타깃 ≥ 44px. 정답/오답을 색에만 의존하지 말 것(체크/엑스 아이콘 동반 — 이미 반영됨).
